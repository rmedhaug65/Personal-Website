Update

I have yet to move my hosting for this website from Github to S3 and CloudFront. In the meantime, please go to my new website to check out my weather station project.

The goal of the project is to be able to remotely measure rain and wind speed in any random field. Typically, one places rain gauges in various areas, then periodically checks them. My idea was to create a digital rain gauge that I would be able to check on my phone. The project isn't quite yet completed, but so far I am able to measure temperature and humidity remotely on my website.

The way this happens is that I have programmed an ESP32 microcontroller, using Arudino's IDE, to measure temperatuer and humidity via a DHT11 sensor, along with latitude and longitude via GPS, then upload that data into a database at AWS. Once the data is uploaded, the website pulls the most recent data from the database for each unique sensor, then plots it on the map using the latitude and longitude of that specific sensor.

The purpose of the project was to get more familiar with AWS, IoT devices, and what building a "pipeline" looks like for developers. I plan on building out the website further and possibly adding an iOS/Android app. I would also at some stage like to get more familiar with orchestrating containers and the monitoring and security surrounding them, so a web app might be in the works at some stage down the road.

Here is a picture of my little beauty :') - the casing was designed in Fusion 360 to fit the dimensions of the solar panel, then printed on my 3D printer. I have been having some issues with maintaining a WiFi connection as the solar panel charges the battery, so I ahve tried attaching a capacitor to maintain the voltage supplying the ESP32. Unfortunately, my electrical enginnering skills are pretty subpar (an understatement), so I apologize in advance if the capacitor is redundant or doesn't make sense!

For the project, I am using a LILYGO ESP32 microcontroller and a DHT11 sensor to measure temperature and humidity data. I am sending the data to Amazon's SQL database (RDS), then using API gateway, in combination with Lambda, to pull the data from RDS into my webpage. For the webpage, the sensor data that is getting pulled combines with Google Map's API to plot the coordinates and display the data.
